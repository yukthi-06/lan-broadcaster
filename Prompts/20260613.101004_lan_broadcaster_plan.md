# LAN Broadcaster Implementation Plan

We will create a multi-threaded LAN communication application using Free Pascal and Lazarus LCL, integrating Indy components (`TIdUDPServer`, `TIdUDPClient`, `TIdTCPServer`, `TIdTCPClient`).

## User Review Required

> [!IMPORTANT]
> The application is built for Windows only.
> It requires the `indylaz` package installed/available in the Lazarus IDE environment to compile successfully.
> Files are transferred over TCP port 5002 in chunk sizes of 64KB to handle files up to and exceeding 1 GB without memory strain.

## Proposed Changes

We will create a clean modular folder structure and implement the following files:

* `LANBroadcaster.lpr` - Project source file.
* `LANBroadcaster.lpi` - Project configuration file.
* `NetworkTypes.pas` - Holds network records, constants, and event definitions.
* `Utilities.pas` - Contains functions for system/network helpers (IP retrieval, Computer Name, directory handling).
* `DiscoveryManager.pas` - Manages background threads for UDP broadcast/hearbeat, and keeping track of online users.
* `MessageManager.pas` - Manages TCP server/client connections for text messages.
* `FileTransferManager.pas` - Manages TCP server/client connections for sending and receiving files in chunks.
* `MainForm.pas` / `MainForm.lfm` - Modern LCL main form layout.

### Component Design

#### [NEW] [NetworkTypes.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/NetworkTypes.pas)
Defines:
- Port constants (`UDP_PORT = 5000`, `TCP_MSG_PORT = 5001`, `TCP_FILE_PORT = 5002`).
- `TOnlineUser` record: `HostName`, `IPAddress`, `LastSeen: TDateTime`.
- Callback and delegate types for UI reporting.

#### [NEW] [Utilities.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/Utilities.pas)
Functions to:
- Get local machine name.
- Query local network IP address using WinSock/Indy.
- Safely check/create `.\ReceivedFiles\` folder.
- Format bytes size representation (KB, MB, GB).

#### [NEW] [DiscoveryManager.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/DiscoveryManager.pas)
- Implements `TDiscoveryManager` class.
- Launches a thread to broadcast `ONLINE|MachineName|IP` to `255.255.255.255` on UDP Port 5000 at regular intervals (e.g., every 5 seconds).
- Listens using `TIdUDPServer` on Port 5000.
- When `ONLINE` message received, updates internal thread-safe list of active users.
- A pruning timer/thread removes users inactive for > 15 seconds.
- Triggers synchronized events to UI for User Joined, User Left, List Updated.

#### [NEW] [MessageManager.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/MessageManager.pas)
- Implements `TMessageManager` class.
- Uses `TIdTCPServer` on Port 5001 to listen for incoming chat messages.
- Parses `MSG|MachineName|Timestamp|Text` format.
- Connects using `TIdTCPClient` to send messages to all active clients sequentially (in a worker thread, to prevent UI freeze).

#### [NEW] [FileTransferManager.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/FileTransferManager.pas)
- Implements `TFileTransferManager` class.
- Uses `TIdTCPServer` on Port 5002 to receive files.
- First receives metadata `FILE|MachineName|FileName|FileSize`.
- Then streams the file into `.\ReceivedFiles\<FileName>` using a binary stream in 64KB chunks.
- Emits download progress event to GUI.
- Uses `TIdTCPClient` to connect to target IP(s) on Port 5002 and send files in 64KB chunks, reporting upload progress.

#### [NEW] [MainForm.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/MainForm.pas) / [MainForm.lfm](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/MainForm.lfm)
- Modern layout with `TListView` for Online Users (IP, Host Name, Status).
- Chat display using `TMemo` (or standard rich controls if preferred, using Memo for stability).
- Input field and [Send Message], [Attach File] buttons.
- `TListView` for active uploads/downloads showing progress bar or percentage.
- `TStatusBar` displaying Local IP, client count, and status messages.
- Handles events from all three managers in a thread-safe manner using `TThread.Queue` or `TThread.Synchronize`.
