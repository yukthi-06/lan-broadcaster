# LAN Broadcaster Icon Resource Implementation Plan

We will fix the invisible tray icon by assigning an icon image to both the application and the `TTrayIcon` components. We will compile `LanBroadcaster.png` as a resource using an `.rc` file and load it at runtime.

## User Review Required

> [!IMPORTANT]
> - A new file [resource.rc](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/resource.rc) will be created to define the PNG resource.
> - The resource will be compiled and linked using FPC's built-in `{$R resource.rc}` compiler directive.
> - Both the Application Window Icon and System Tray Icon will display `LanBroadcaster.png`.

## Proposed Changes

### [NEW] [resource.rc](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/resource.rc)
Define the resource mapping for the PNG file:
```rc
LANBROADCASTER_PNG RCDATA "LanBroadcaster.png"
```

### [MODIFY] [MainForm.pas](file:///x:/Projects_X/0_Active/4_Lazarus/AI_Helped/LAN_Broadcaster_Lazarus_GIT/MainForm.pas)
- Add `LCLType` to the interface `uses` clause to obtain the `RT_RCDATA` constant.
- Add compiler directive `{$R resource.rc}` to link the resource file.
- Inside `FormCreate`, load the PNG from the compiled resource using `TResourceStream` and assign it to `Application.Icon` and `TrayIcon.Icon`.
