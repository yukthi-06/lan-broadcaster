# User Prompt
```
Create a complete desktop application in Lazarus (Free Pascal) called "LAN Broadcaster".

Requirements:

* Platform: Windows only.
* Framework: Lazarus LCL.
* Language: Object Pascal (Free Pascal).
* Produce a fully working Lazarus project with all units and forms.
* Use clean Object Pascal coding standards.
* Use classes and separation of concerns.
* Add comments explaining important sections.

Application Purpose:

The application allows any user on a local network (LAN) to send messages and file attachments to all other computers running the same application.

Networking Requirements:

1. Automatic Discovery

   * Use UDP Broadcast for discovery.
   * All running instances must automatically discover each other.
   * No manual IP entry.
   * Display online users in a list.
   * Show:

     * Computer Name
     * IP Address
     * Status (Online)

2. Messaging

   * Allow sending a text message.
   * Message is delivered to all discovered clients.
   * Display incoming messages in a chat/log window.
   * Include:

     * Timestamp
     * Sender Machine Name
     * Message Text

3. File Broadcasting

   * Allow selecting a file using OpenDialog.
   * Broadcast file metadata first.
   * Transfer files using TCP.
   * Support files up to at least 1 GB.
   * Transfer files in chunks (do not load entire file into memory).
   * Display transfer progress.
   * Save received files into:
     .\ReceivedFiles\
   * Automatically create the folder if it does not exist.

4. Reliability

   * Use TCP for messages and file transfers.
   * Use UDP only for discovery.
   * Handle disconnections gracefully.
   * Rebuild online user list automatically.
   * Prevent application crashes from network errors.
   * Add exception handling throughout networking code.

5. Multithreading

   * Networking must run in worker threads.
   * UI must never freeze.
   * Use TThread and thread-safe UI updates.
   * Use Synchronize or Queue when updating controls.

GUI Requirements:

Create a modern professional interface.

Main Window Layout:

---

## | LAN Broadcaster                                  |

| Online Users |
| ------------ |
| ListView     |
|              |

---

| Messages         |
| ---------------- |
| RichMemo or Memo |
|                  |

---

| Message Input                                    |
| [___________________________________________]    |
|                                                   |
| [Send Message] [Attach File]                     |
----------------------------------------------------

| Transfers              |
| ---------------------- |
| ListView with progress |

---

## | Status Bar                                       |

Features:

* No Dark theme.
* Resizable window.
* Status bar showing:

  * Local IP
  * Number of connected clients
  * Current status
* Message log with timestamps.
* Transfer log.
* Progress bars for uploads and downloads.
* Notification when a new file arrives.
* Notification when a user comes online/offline.

Implementation Details:

Use:

* TIdUDPServer
* TIdUDPClient
* TIdTCPServer
* TIdTCPClient

or equivalent Lazarus-compatible Indy components.

Suggested Ports:

* UDP Discovery: 5000
* TCP Messaging: 5001
* TCP File Transfer: 5002

Protocol Design:

Discovery:
DISCOVER
ONLINE|MachineName|IP

Message:
MSG|MachineName|Timestamp|Text

File Announcement:
FILE|MachineName|FileName|FileSize

Transfer:
Chunked TCP streaming with progress reporting.

Code Quality:

* Create separate units:

  * MainForm
  * DiscoveryManager
  * MessageManager
  * FileTransferManager
  * NetworkTypes
  * Utilities

* Avoid putting all code in one form unit.

* Follow object-oriented design.

* Use records/classes where appropriate.

* Include all source code.

* Include .lfm form definitions.

* Include project file.

* Ensure the project compiles in Lazarus without missing references.

Deliverables:

Generate the complete Lazarus project source code, all units, all forms, and instructions for building and running the application.
```
